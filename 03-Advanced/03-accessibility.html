HTML Accessibility (ARIA)

Web accessibility ensures that websites are usable by everyone, including people with disabilities. ARIA (Accessible Rich Internet Applications) provides additional semantics to help assistive technologies understand your content.

Why Accessibility Matters

  • 15% of the world's population has some form of disability
  • Legal requirement in many countries (ADA, Section 508)
  • Better SEO - search engines favor accessible sites
  • Improved UX for everyone, not just people with disabilities
  • Wider audience reach - more users can access your content

Semantic HTML First

Always use semantic HTML before adding ARIA. Native HTML elements have built-in accessibility.

Alt Text for Images

Always include meaningful alt text for images. Screen readers announce this text to users.

‚ùå Bad

<img src="photo.jpg">
<img src="logo.jpg" alt="image">

‚úÖ Good

<img src="photo.jpg" alt="Team celebrating project launch">
<img src="logo.jpg" alt="Company logo">
Alt Text Guidelines:
  • Describe the content and function of the image
  • Keep it concise (typically under 125 characters)
  • Don't start with "image of" or "picture of"
  • Use alt="" for decorative images
  • Include text if the image contains text

Form Accessibility

Always Label Form Inputs

‚ùå Bad

<input type="text" placeholder="Name">

‚úÖ Good

<label for="name">Name:</label>
<input type="text" id="name" name="name">

Provide Error Messages

<label for="email">Email:</label>
<input type="email" 
       id="email" 
       name="email" 
       aria-describedby="email-error"
       aria-invalid="true">
<span id="email-error" role="alert">
    Please enter a valid email address
</span>

Keyboard Navigation

All interactive elements must be keyboard accessible.

Interactive Demo

Keyboard Navigation Requirements:
  • Tab - Move to next interactive element
  • Shift + Tab - Move to previous element
  • Enter or Space - Activate buttons/links
  • Visible focus indicator (outline) on focused elements
  • Logical tab order (follows visual layout)

Focus Management

/* Always show focus indicators */
button:focus {
    outline: 3px solid #ffc107;
    outline-offset: 2px;
}

/* Never do this */
*:focus {
    outline: none; /* ‚ùå Removes accessibility */
}

ARIA Roles

ARIA roles define what an element is or does.

Common ARIA Roles:

<nav role="navigation">...</nav>
<main role="main">...</main>
<div role="button">...</div>
<div role="alert">...</div>
<div role="dialog">...</div>
<div role="tooltip">...</div>
Note: HTML5 semantic elements already have implicit roles. Only add ARIA roles when using non-semantic elements.

Heading Structure

<h1>Main Title</h1>
<h3>Subtitle</h3> <!-- Skips h2 -->
<h2>Section</h2>

‚úÖ Good

<h1>Main Title</h1>
<h2>Subtitle</h2>
<h3>Section</h3>

Link Accessibility

‚ùå Bad

<a href="article.html">Click here</a>
<a href="page.html">Read more</a>

‚úÖ Good

<a href="article.html">Read the full article about accessibility</a>
<a href="page.html">Read more about ARIA labels</a>
Link Best Practices:
  • Link text should make sense out of context
  • Avoid generic text like "click here" or "read more"
  • Indicate when links open in new windows
  • Use descriptive text that explains the destination

Color and Contrast

  • WCAG AA: Contrast ratio of at least 4.5:1 for normal text
  • WCAG AAA: Contrast ratio of at least 7:1 for normal text
  • Don't rely on color alone to convey information
  • Use patterns or icons in addition to color
