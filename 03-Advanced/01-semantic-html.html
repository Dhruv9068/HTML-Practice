What is Semantic HTML?

Semantic HTML uses HTML elements that have meaning. Instead of using generic <div> and <span> tags, semantic elements clearly describe their purpose to both the browser and the developer.

Why Semantic HTML Matters

  • Accessibility: Screen readers can better understand the page structure
  • SEO: Search engines can better index your content
  • Maintainability: Code is easier to read and understand
  • Consistency: Standard elements mean consistent behavior

Common Semantic HTML5 Elements

Page Structure Elements

  • <header> - Introductory content or navigation
  • <nav> - Navigation links
  • <main> - Main content of the page (only one per page)
  • <article> - Self-contained content
  • <section> - Thematic grouping of content
  • <aside> - Content aside from main content (sidebars)
  • <footer> - Footer for document or section

Content Elements

  • <figure> - Self-contained content like images
  • <figcaption> - Caption for figure
  • <time> - Date/time information
  • <mark> - Highlighted text
  • <details> - Disclosure widget
  • <summary> - Summary for details element

Practical Examples

Example 1: Blog Post Structure

<article>
    <header>
        <h1>Blog Post Title</h1>
        <time datetime="2024-01-15">January 15, 2024</time>
    </header>
    
    <section>
        <h2>Introduction</h2>
        <p>Content here...</p>
    </section>
    
    <footer>
        <p>Author: John Doe</p>
    </footer>
</article>

Example 2: Figure with Caption

<figure>
    <img src="diagram.jpg" alt="Description">
    <figcaption>Figure 1: Semantic HTML5 structure</figcaption>
</figure>

üí° Quick Tips

  • Use <header> for introductory content
  • Use <nav> for navigation menus
  • Use <main> for the primary content
  • Use <article> for standalone content
  • Use <section> to group related content
  • Use <aside> for sidebars or related info
  • Use <footer> for footer information
Best Practices:
  • Always prefer semantic elements over generic divs
  • Use only one <main> element per page
  • Don't nest <article> inside <address>
  • Each <section> should have a heading
  • Use <article> for content that makes sense on its own
